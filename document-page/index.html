Introduction

Git is a distributed version control system commonly used for tracking changes in source code during software development. Created by Linus Torvalds in 2005, Git has become the de facto standard for version control, allowing multiple developers to work on projects simultaneously while keeping track of every change made.

What is Version Control?

Version control is a system that allows developers to manage changes to code over time. It enables the following:

  • Tracking Changes: Keep a history of every change made to the project files.
  • Collaboration: Allows multiple people to work on a project without overwriting each other’s changes.
  • Reverting: Provides a way to go back to a previous state in case of errors or unwanted changes.

There are two main types of version control systems:

  • Centralized Version Control (CVCS): A single central repository, like SVN.
  • Distributed Version Control (DVCS): Each user has a complete copy of the project history, like Git.
Understanding Git

Git is a distributed version control system that keeps track of code changes. Some key concepts include:

  • Repository: A directory that contains all project files and a .git folder with metadata.
  • Commit: A snapshot of changes in the repository.
  • Branch: A parallel version of a repository.
  • Merge: The process of integrating changes from one branch to another.

Git’s distributed nature allows each developer to have their own copy of the project, making it easy to work offline and preventing data loss.

Basic Git Commands

Below are some essential Git commands:

  • git init: Initializes a new Git repository in the current directory.
  • git clone: Copies an existing repository from a remote location.
  • git add: Stages files to be committed.
  • git commit -m "message": Commits the staged changes with a message.
  • git status: Displays the status of the working directory.
  • git log: Shows a history of commits in the repository.

Example:

git init
git add .
git commit -m "Initial commit"
Branching and Merging

Branching is a powerful feature in Git that allows you to work on different versions of a project simultaneously.

  • Creating a Branch:Use git branch branch-name to create a new branch.
  • Switching Branches:Use git checkout branch-name to switch to an existing branch.
  • Merging Branches:Integrate changes from one branch to another with git merge branch-name.
Working with Remote Repositories

Remote repositories are hosted on the internet or an intranet. Git allows you to push and pull code to/from these remote repositories.

  • git remote add origin url: Links your local repo to a remote repo.
  • git push origin branch-name: Pushes commits to the remote repository.
  • git pull origin branch-name: Pulls changes from the remote repository.

Example

git remote add origin https://github.com/username/repository.git
git branch -M main
git push -u origin main
Common Workflows

Here are some common Git workflows used in team environments:

  • Feature Branch Workflow: Each feature or bug fix is developed in a separate branch.
  • Gitflow Workflow: Uses a specific branch structure with main, develop, feature, and hotfix branches.
  • Forking Workflow: Ideal for open source projects where contributors fork the repository, make changes, and submit pull requests.
